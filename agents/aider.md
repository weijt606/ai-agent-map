## When To Pick It

- You already live in the terminal and want AI help without moving into a whole new IDE.
- You want git diffs and commits to stay central to how the agent makes changes.
- You want broad model choice, including local models, instead of one tightly bundled provider path.

## When Not To Pick It

- You want a polished team UI or a manager-friendly cloud dashboard.
- You want fully detached background delegation as the main workflow.
- You need a broader multi-agent platform instead of a terminal pair programmer.

## Capability Shape

| Dimension | Assessment | Notes |
| --- | --- | --- |
| Terminal workflow | Very strong | The terminal is the natural home |
| Git integration | Very strong | A major differentiator |
| Model flexibility | Strong | Works with many hosted and local models |
| IDE adjacency | Medium | Possible, but not the core operating model |
| Explicit approval | Medium | Control comes more from diffs and git than per-action prompts |

## Operating Cost

Complexity is Medium. Installation is straightforward, but good results still depend on model choice, API setup, and whether your team is comfortable with a terminal-led workflow.

## Bottom Line

Aider is one of the clearest open-source answers to “I want AI pair programming, but I do not want to leave the terminal or give up git discipline.”
