## When To Pick It

- You want a serious open-source path for software engineering agents.
- You want CLI and GUI today with room to move toward cloud or enterprise later.
- You want optionality between self-hosted and hosted operation.

## When Not To Pick It

- You need very low setup and ops burden.
- You only want a lightweight editor-native agent.
- You only need an orchestration framework, not a finished agent.

## Capability Shape

| Dimension | Assessment | Notes |
| --- | --- | --- |
| Self-hosting | Strong | A major part of the value proposition |
| CLI workflow | Strong | Good fit for developer-first environments |
| GUI workflow | Strong | Not restricted to CLI-only usage |
| Cloud deployment | Strong | The route options are relatively broad |
| MCP / extensibility | Medium | Extensible, but not the only reason to choose it |

## Operating Cost

Complexity is Medium. Its value is high because it really runs. Its cost is also real because Docker, model setup, and related environment work are part of the story.

## Bottom Line

If you are seriously asking whether open-source agents can replace part of a closed-source coding workflow, OpenHands belongs near the front of the list.
