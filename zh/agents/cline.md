一句话：Cline 适合那种想让 agent 很能干，但又不想把控制权完全交出去的人。

## 一眼判断

| 项目 | 结论 |
| --- | --- |
| 厂商 | Cline |
| 路线 | approval-first 编辑器内 coding agent |
| 是否开源 | 是 |
| 最适合 | 强控制、本地编辑器工作流 |
| 主要代价 | 高频审批会带来摩擦 |
| 官方入口 | https://docs.cline.bot/ |

## 什么时候选它

- 你希望每一步关键动作都先看一眼再放行。
- 你想在编辑器里同时改代码、跑命令、调浏览器。
- 你准备把 MCP 接进来，扩展专用工具能力。

## 什么时候别选

- 你想把任务完全后台化，自己尽量少参与。
- 你不接受审批打断工作节奏。
- 你要的是面向团队统一管理的云端代理平台。

## 能力侧写

| 维度 | 判断 | 备注 |
| --- | --- | --- |
| File editing | 强 | 在编辑器里动手能力很完整 |
| Terminal execution | 强 | 适合边跑边看输出 |
| Browser use | 强 | 做前端和 UI 调试时很有价值 |
| MCP | 强 | 工具扩展能力是亮点之一 |
| Human approval | 很强 | 它最核心的差异化就在这里 |

## 使用代价

复杂度是 Medium。装上不难，真正的成本是你愿不愿意接受“强能力 + 强审批”这种操作节奏。

## 最后一句

Cline 不是在追求完全自治，它是在追求“agent 很强，但人始终握着方向盘”。
