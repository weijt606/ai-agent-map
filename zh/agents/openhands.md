一句话：OpenHands 是当前最值得认真评估的开源软件工程 agent 之一，因为它不是 demo，而是一套真能跑起来的系统。

## 一眼判断

| 项目 | 结论 |
| --- | --- |
| 厂商 | OpenHands |
| 路线 | 开源软件工程执行 agent |
| 是否开源 | 是 |
| 最适合 | 想走开源路线，但又不只要一个玩具项目 |
| 主要代价 | Docker、模型配置和运行环境成本更高 |
| 官方入口 | https://github.com/OpenHands/OpenHands |

## 什么时候选它

- 你想用开源方式评估软件工程 agent 的真实上限。
- 你既想有 CLI，也想有 GUI，还想保留未来接 cloud 或 enterprise 的路径。
- 你希望在自托管和托管之间保留选择权。

## 什么时候别选

- 你要求极低运维和极轻安装成本。
- 你只想要编辑器里的轻量 agent。
- 你真正只需要 orchestration framework，而不是成品 agent。

## 能力侧写

| 维度 | 判断 | 备注 |
| --- | --- | --- |
| Self-hosting | 强 | 是重要价值之一 |
| CLI workflow | 强 | 适合开发者环境 |
| GUI workflow | 强 | 不是只有命令行可用 |
| Cloud deployment | 强 | 路线比较完整 |
| MCP / extensibility | 中 | 能扩，但不是唯一卖点 |

## 使用代价

复杂度是 Medium。它的价值很大，因为它真能跑；它的成本也很真实，因为 Docker、模型、搜索配置这些都要你自己处理。

## 最后一句

如果你在认真看“开源能不能替代一部分闭源 coding agent”，OpenHands 应该放在前排。
